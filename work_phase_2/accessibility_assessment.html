Accessibility Assessment

Drawing from the principals of the Web Content Accessibility Guidelines (WCAG) 2.2, and assessing these against our client and user needs, we have identified twelve criterion that the website will need to succeed at.

Success Criterion 1.1.1 Alternative Text for Non-Text Content

Rationale

The website will contain non-text content, such as images and diagrams.

Relevant Needs/Features

  • Accessibility
  • Photos and diagrama

Success Criterion 1.3.2 Meaningful Sequence

Rationale

The website will contain contain that is meaningful, such as topics threads intended to be read sequentially.

Relevant Needs/Features

  • Accessibility
  • Videos

Success Criterion 1.4.1 Avoid Color-Only Cues

Rationale

The website will use colour to convey meaning, therefore an additional mode of perception (such as text) will be used in conjunction with colour.

Relevant Needs/Features

  • Accessibility

Success Criterion 1.4.3 Perceivable Text Contrast

Rationale

As the website will contain text, measures will be taken to ensure the minimum contrast meets this criterion.

Relevant Needs/Features

  • Accessibility (Client Need)

Success Criterion 1.4.4 Resize Text without Loss

Rationale:

As the website will contain text, measures will be taken to ensure the text can be resized to meet this criterion.

Relevant Needs/Features

  • Accessibility (Client Need)

Success Criterion 2.1.1 Keyboard Navigability

Rationale

The website will be navigable through keyboard only input for users with accessibility considerations or preferences.

Relevant Needs/Features

  • Accessibility
  • User-friendly

Success Criterion 2.4.7 Focus Visible

Rationale

The website will be navigable through keyboard only input, therefore the keyboard focus indicator must be visible.

Relevant Needs/Features

  • Accessibility
  • User preferring mouse-free navigation.
  • User-friendly

Success Criterion 3.1.5 Reading Level

Rationale

The website audience will be diverse in age and reading level, therefore it will be required to supplement advanced language.

Relevant Needs/Features

  • Accessibility
  • Inclusivity
  • Diversity

Success Criterion 3.2.4 Consistent Identification (Layout & UI Behaviour)

Rationale

There will be components that have the same functionality within pages, such as a commercial activity flag button.

Relevant Needs/Features

  • Accessibility
  • User-friendly
  • Icons
  • Terms

Success Criterion 3.3.1 Error Identification & Suggestions

Rationale

The website will contain user input fields in the newsletter signup form.

Relevant Needs/Features

  • Newsletter Signup Form
  • User-friendly
  • Accessibility

Success Criterion 3.3.3 Error Suggestion

Rationale

The website will contain user input fields in the newsletter signup form.

Relevant Needs/Features

  • Newsletter Signup Form
  • User-friendly
  • Accessibility
  • Ethical Design

Success Criterion 3.3.4 Error Prevention & Recovery

Rationale

The website may contain a user registration for the forum and therefore the website will need to manage user-controllable data.

Relevant Needs/Features

  • Newsletter Signup Form
  • Forum
  • Accessibility
  • Ethical Design
